기대효과

  • - 블로그에 필요한 기능을 알게됩니다.

  • - 블로그 생성도구를 만들면서 사용한 기술을 알게됩니다.

  • - 글보다 소스코드를 먼저 보고 싶으신 분은 github.com/jetsbee/jetsbee.github.io 를 참고해 주세요.

안녕하세요. 블로그 생성도구를 만들게 되었습니다.

블로그를 시작하기 위해서 블로그 생성도구를 꼭 만들 필요는 없습니다.

그래도 저처럼 특정 서비스에 종속되고 싶지 않으면서 간편하게 블로그를 운영하고 싶으신 분이 계시다면, 이 연재글이 도움이 될지도 모르겠습니다.

블로그에 필요한 기능

아래는 제가 블로그에 필요한 기능입니다. 아직 구현하지 않은 부분은 하나씩 만들어 갈 예정입니다.

  • - 백엔드 운영이 필요없는 컨텐츠 관리 시스템(WYSIWYG 에디터 등)

- 컨텐츠를 볼 수 있는 웹사이트

  • - SEO(검색 엔진 최적화 - Meta tags, JSON-LD, Open Graph, Sitemaps, Robots, 검색엔진 등록, RSS, JSON Feed 등)

  • - 웹 로그 분석(방문자 특성 분석, 방문자 행동 분석 - GA)

  • - 페이지네이션 or 무한스크롤

  • - 컨텐츠 태깅(필터기능)

  • - 검색

  • - 댓글

  • - ToC(Table of Contents, 목차)

기능을 만들어 가면서 블로그 생성도구 제작기를 계속 올리도록 하겠습니다.

블로그 생성도구를 만들면서 사용한 기술

1. Turborepo

웹페이지, 에디터, 컨텐츠 등을 별도 프로젝트로 관리하기 위해 모노레포를 도입했습니다.

모노레포를 관리하기 Turborepo 를 도입하였고, 그 결과 더욱 유연한 아키텍처를 가질 수 있었습니다.

자세한 내용은 다음 연재글에서 다룰 예정입니다.

2. Next.js

아주 강력한 React meta framework 입니다. 이미 관련 내용이 아주 많습니다.

자세한 내용을 보고 싶으시면 공식문서를 추천합니다.

저는 Next.js app router 를 사용합니다.

3. Tailwind CSS

안정적인 디자인 시스템 구축, 생산성 등의 이유로 도입하게 되었습니다.

저는 shadcn/ui 와 같이 사용합니다.

4. 기타

WYSIWYG 에디터 구현을 도와주는 tiptap 등 기타 몇가지 라이브러리가 있습니다.

자세한 내용은 앞으로의 연재글에서 다룰 예정입니다.
